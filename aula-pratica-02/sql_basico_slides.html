O que é
SQL?

Structured Query Language — é a linguagem usada para conversar com bancos de dados relacionais. Com ela, você diz ao banco o que quer fazer com os dados: buscar, criar, alterar ou apagar.

Não é uma linguagem de programação no sentido tradicional. É uma linguagem declarativa — você descreve o que quer, e o banco de dados descobre como fazer.

Data Training Program 03 / 10
Create

CREATE
& INSERT

CREATE TABLE define a estrutura — os nomes das colunas e os tipos de dado. INSERT INTO adiciona linhas nessa estrutura.

  • Você define as colunas e o tipo de cada uma
  • Tipos comuns: TEXT, INTEGER, FLOAT, DATE
  • Sem uma tabela, não tem onde guardar o dado
SQL CREATE TABLE clima (
  cidade   TEXT,
  data     DATE,
  temp_max FLOAT,
  chuva_mm FLOAT
);
INSERT INTO clima
  (cidade, data, temp_max, chuva_mm)
VALUES
  ('São Paulo', '2024-01-15', 31.2, 12.5);

Data Training Program 05 / 10
Read

SELECT

O comando mais usado em SQL. Você consulta os dados sem alterar nada — só lê. É por onde começa qualquer análise.

  • SELECT * traz todas as colunas
  • WHERE filtra as linhas por condição
  • ORDER BY ordena o resultado
  • LIMIT restringe quantas linhas retornam
SQL -- todas as colunas
SELECT * FROM clima;
-- colunas específicas + filtro
SELECT cidade, temp_max
FROM clima
WHERE temp_max > 30;
-- ordenar e limitar
SELECT cidade, chuva_mm
FROM clima
ORDER BY chuva_mm DESC
LIMIT 3;

Data Training Program 07 / 10
Update

UPDATE

Modifica o valor de colunas em linhas que já existem. Sempre use com WHERE — sem ele, você altera a tabela inteira.

  • SET define qual coluna muda e o novo valor
  • WHERE determina quais linhas são afetadas
  • Sem WHERE, todas as linhas são atualizadas
SQL -- corrigir um valor específico
UPDATE clima
SET temp_max = 29.8
WHERE cidade = 'São Paulo'
  AND data = '2024-01-15';
-- sem WHERE: altera tudo (cuidado!)
UPDATE clima
SET chuva_mm = 0;
-- zeraria TODAS as linhas

Data Training Program 09 / 10
Delete

DELETE
& DROP

DELETE remove linhas de uma tabela. DROP TABLE apaga a tabela inteira — estrutura e dados. Ambos são irreversíveis sem backup.

  • DELETE + WHERE remove linhas específicas
  • DELETE sem WHERE esvazia a tabela
  • DROP TABLE apaga a tabela por completo
SQL -- remover uma linha específica
DELETE FROM clima
WHERE cidade = 'Rio de Janeiro';
-- esvaziar a tabela (sem WHERE)
DELETE FROM clima;
-- remove TODAS as linhas
-- apagar a tabela completamente
DROP TABLE clima;
